What is the CPU?

The CPU, or Central Processing Unit, is the primary component of a computer that performs most of the processing inside the computer. It’s often referred to as the “brain” of the computer.The processor (also called the Central Processing Unit or CPU) has a number of different components, each with their own role to perform:

  1. Control Unit
  2. Buses
  3. Arithmetic-Logic Unit
  4. Dedicated Registers

Parts of the CPU

Control Unit

This controls the flow of data within the system. It monitors communications between the hardware attached to the computer, controls the input and output of data, checks that signals have been delivered successfully, and ensures that data goes to the correct place at the correct time

The Control Unit (CU) of a computer operates by following a series of steps. It begins by fetching instructions from the main memory (RAM). These instructions are then decoded to determine the specific operations that need to be performed. The CU directs the appropriate components within the CPU to carry out these operations. It also manages and controls input, output, and storage. For instance, it sends control signals to the Arithmetic Logic Unit (ALU) for performing arithmetic and logical calculations as per the requirement. After calculations, it sends it to the register set to save or hold it temporarily. The CU ensures that instructions are executed in the correct order and synchronizes the activities of various components within the CPU to maintain proper operation.

Bus

The CPU bus, often referred to as “the bus”, is a pathway composed of cables and connectors used to carry data between a computer microprocessor and the main memory1. It’s essentially a series of lines that connect the processor to another part of the computer’s architecture, such as cache memory or main memory2. The bus provides a communication path for the data and control signals moving between the major components of the computer system1. There are three main types of buses in a CPU:

Control Bus: The control bus in a CPU is a pathway used to transmit control signals between the CPU and other devices within the computer. These signals coordinate and regulate the actions of the computer's hardware, ensuring smooth operation.
Address Bus: This is uni-directional, meaning it passes an address one way, from the CPU to RAM. The sole purpose of an address bus is to identify the address of the location in cache or main memory that is to be read from or written to.

Data Bus: This is bi-directional, meaning it can carry data to main memory from the processor and vice versa. The data bus transfers data to/from the address that is held on the address bus.

The amount of data that can be carried by the data bus depends on the word size, which describes the width of the data bus. For instance, new processors usually have a word size of 8 bytes (64 bits), allowing for 64 bits to be transferred during each cycle.

Arithmetic-Logic Unit

The Arithmetic Logic Unit (ALU) is a key component of the computer’s central processing unit (CPU). It performs arithmetic and bitwise operations on binary numbers. The ALU receives data inputs and a code indicating the operation to be performed, then executes these operations which can include addition, subtraction, multiplication, and division. It also performs logical operations and decision-making tasks. The ALU acts as a gateway between primary memory and secondary storage, with data transferred between them passing through the ALU. The results of these operations are then outputted from the ALU. The ALU can also compare values and determine if values in registers are identical. It is coordinated by the Control Unit (CU), which employs the ALU as required to fetch and decode instructions, read data from storage, execute commands, and possibly write registers back.

Registers

Registers are small storage areas within the CPU where data is stored and manipulated during processing. They are a critical part of a computer’s central processing unit (CPU) and contribute to the speed and efficiency of a computer’s operation.The Program Counter (PC) holds the address of the next instruction, the Current Instruction Register (CIR) holds the current instruction, the Memory Address Register (MAR) holds the address of the current instruction, the Memory Data Register (MDR) holds data fetched from memory, and the Accumulator stores results of calculations.

Executing Instructions

Processors operate in defined stages that are used to carry out program instructions. The process is repeated over and over again for each instruction in a program. Below are the steps in the Fetch-Execute Cycle:

  1. The address of the next instruction is copied from the PC to the Memory Address Register (MAR)
  2. The instruction held at that address is copied to the Memory Data Register (MDR)
  3. Simultaneously, the contents of the Program Counter (PC) are incremented
  4. The contents of the MDR are copied to the current Instruction Register (CIR)
  5. The instruction held in the CIR is decoded
  6. It is split into operand and opcode to determine the type of instruction it is. Additional data, if required, is fetched from memory…
  7. ...and passed to the accumulator The opcode specifies the operation that is to be carried out. The operand holds either:
    The address of the data to be used, which is then copied to the MAR, or
    The actual data to be operated on, which is passed to the MDR
  8. The instruction is executed and the result held in accumulator or stored in memory
